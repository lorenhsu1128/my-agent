## 🎯 設定檔與行為對應表

```
settings.jsonc 欄位                  ──────→  觸發行為
─────────────────────────────────────────────────────────────
autoMemoryEnabled: true             → 自動學習用戶偏好/修正
autoDreamEnabled: true              → 定期 (24h/5 sessions) 記憶整合
skillImprovementTurnBatch: 5        → 每 5 個 turn 檢查 skill 改進
memoryNudgeTurnBatch: 8             → 每 8 個 turn 偵測用戶偏好
skillCreationToolUseThreshold: 15   → ≥15 個 tool 呼叫後生成 skill
sessionReviewMinToolUses: 15        → ≥15 個 tool 呼叫才 review
sessionReviewMinIntervalHours: 2    → 2 小時後才進行第二次 review
autoDreamMinHours: 24               → 24 小時才觸發記憶整合
autoDreamMinSessions: 5             → 5 個 session 後才觸發整合
includeCoAuthoredBy: true           → Git commit 包含標記
cleanupPeriodDays: 30               → 30 天後清理舊 session
```

## 📝 記憶檔案實際範例

### feedback 類型記憶（最常見）

```markdown
---
name: feedback_test_mocking
description: 測試必須連接真實資料庫
type: feedback
---

**規則：** Integration tests 必須連接真實資料庫，不得 mock

**Why:**
上季有 mocked 測試通過但 prod migration 失敗，導致數據不一致

**How to apply:**
- 所有資料庫測試不得使用 mock
- 即使測試環境也必須使用真實 DB 連接
- 如果測試需要隔離，使用 transaction 而非 mock
```

### user 類型記憶（全局優先）

```markdown
---
name: language_preference
description: 用戶語言偏好
type: user
---

Always reply in 繁體中文 (Traditional Chinese)
```

### project 類型記憶（專案特定）

```markdown
---
name: project_release_freeze
description: 4/25 後非緊急 PR 暫停合併
type: project
---

**事實：** 4/25 後進入 release freeze 週期

**Why:**
Mobile 團隊在準備重要 release，需要穩定版本

**How to apply:**
- 非緊急 bug fix 暫時不要 merge
- 優先處理 blocking 的 critical issues
- 其他 PR 進入 waiting list
```
